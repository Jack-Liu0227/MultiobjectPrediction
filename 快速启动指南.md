# 🚀 快速启动指南

欢迎使用多目标材料性能预测系统！本指南将帮助您在 5 分钟内启动系统。

---

## ⚡ 一键启动（推荐）

### Windows 用户

#### 方法 1：批处理脚本（最简单）

1. **双击运行** `start_all.bat` 文件
2. 等待系统自动检查和启动（约 30 秒）
3. 看到提示后，输入 `Y` 自动打开浏览器
4. 开始使用！

#### 方法 2：PowerShell 脚本（推荐高级用户）

1. 右键点击 `start_all.ps1`，选择"使用 PowerShell 运行"
2. 如果遇到执行策略限制，以管理员身份运行 PowerShell：
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```
3. 然后运行：
   ```powershell
   .\start_all.ps1
   ```

### Linux/Mac 用户

```bash
# 首次使用，添加执行权限
chmod +x start_all.sh

# 启动服务
./start_all.sh

# 停止服务：按 Ctrl+C
```

## 🛑 停止服务

### Windows

**方法 1：** 双击运行 `stop_all.bat`

**方法 2：** 运行 PowerShell 脚本
```powershell
.\stop_all.ps1
```

**方法 3：** 关闭后端和前端的命令行窗口

### Linux/Mac

按 `Ctrl+C` 停止服务

## ⚠️ 首次使用前的准备

### 1. 确认系统要求

- ✅ Python 3.10 或更高版本
- ✅ Node.js 16 或更高版本
- ✅ 至少 4GB 可用内存
- ✅ 至少 2GB 可用磁盘空间

### 2. 下载 RAG 模型

如果您还没有下载 `all-MiniLM-L6-v2` 模型，请按照以下步骤操作：

**方法 1：自动下载（推荐）**

在项目根目录运行：
```bash
python download_model.py
```

**方法 2：手动下载**

1. 访问：https://huggingface.co/sentence-transformers/all-MiniLM-L6-v2
2. 下载所有文件到项目根目录的 `all-MiniLM-L6-v2` 文件夹

### 3. 配置环境变量（可选）

如果您想使用 LLM 功能，需要配置 API 密钥：

1. 复制 `.env.example` 为 `.env`：
   ```bash
   # Windows
   copy .env.example .env
   
   # Linux/Mac
   cp .env.example .env
   ```

2. 编辑 `.env` 文件，填入您的 API 密钥：
   ```env
   DEEPSEEK_API_KEY=your_deepseek_api_key_here
   GEMINI_API_KEY=your_gemini_api_key_here
   ```

**获取 API 密钥：**
- DeepSeek: https://platform.deepseek.com/
- Gemini: https://ai.google.dev/

## 📝 使用流程

1. **上传数据**
   - 访问 http://localhost:3000/prediction
   - 点击"上传 CSV 文件"
   - 选择您的材料数据文件

2. **配置预测**
   - 选择组成列（材料成分）
   - 选择热处理列（工艺参数）
   - 选择目标列（性能指标，2-5 个）
   - 配置 RAG 参数和 LLM 模型

3. **开始预测**
   - 点击"开始预测"
   - 等待预测完成（进度条显示）

4. **查看结果**
   - 查看预测结果表格
   - 查看 Pareto 前沿分析
   - 查看评估指标（R², RMSE, MAE, MAPE）
   - 下载结果 CSV 文件

## 🔍 故障排除

### 问题 1：启动脚本报错 "找不到 Python"

**解决方案：**
1. 确认已安装 Python 3.10+
2. 在命令行运行 `python --version` 验证
3. 如果未安装，访问 https://www.python.org/downloads/ 下载安装

### 问题 2：启动脚本报错 "找不到模型"

**解决方案：**
1. 确认 `all-MiniLM-L6-v2` 文件夹存在于项目根目录
2. 运行 `python download_model.py` 自动下载模型

### 问题 3：端口被占用

**解决方案：**
1. 检查端口 8000 和 3000 是否被其他程序占用
2. 关闭占用端口的程序
3. 或修改配置使用其他端口

### 问题 4：依赖安装失败

**解决方案：**
1. 检查网络连接
2. 使用国内镜像源：
   ```bash
   # Python
   pip install -r backend/requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
   
   # Node.js
   npm config set registry https://registry.npmmirror.com
   cd frontend
   npm install
   ```
